A brief introduction into Noble PHP framework

I built this framework with the need/intent of being a straight forward, no thrills, no magic, purely practical framework. Built out of practical applications, Noble is more of a organizational convention mixed with helping libraries to help you work, rather than an overly complicated environment that makes things harder by trying to do work for you.

Conventions

Organizational conventions are straight foward. They are seperated by functionality typical of a web applicaiton.

The framework is organized in the following

Libraries

The core files of the framework
  • Noble - The main entrance point for the framework, every transaction (http|cli) begins here - Contains path defintions and pointers to libraries
  • Boostrap - Conventions loader, a simple interface for value->key based configurations
  • Template - Defines conventions for interfacing templates with each other and provides inheritance
  • Controllers & Models - The parent class of all controller and models, each extends their parent respectively.
  • Responder - The object which gets passed between all interactions of the framework during a request
  • Route - Provides URL/Cli path handling capabilities, pairs routes to actions and allows for variable path parameters

Templating

The goal of the framework was to allow fully naitive php for templating while maintaining useful behaviors such as template inheritance, extending parent templates, block based templating and block prepending/appending functionality.

When specified to use a template, the framework will include a single starting template and pass all template variables into it.

The starting template will be within the "Template" Library, and will have the ability to extend other templates and blocks

Inside each template, you can use both standard HTML and PHP, PHP must be used normally with >?php and ?< tags;

Blocks

Blocks are overidable section of the template you can reference by name

          /// in the extending template (example. view.home.html)
          

            <?php self::block("title"); ?>
              This is the content of the block
            <?php self::endblock(); ?>
          

          <?php self::extend("html/template.master.html"); ?>

          /// in the base template (template.master.html);
           

            <?php self::block("title"); ?>
              This is the default content of the block
            <?php self::endblock("title"); ?> //// end a block with its own name to display its content, 
            /// normally only done on the top most level of the templates where the block is displayed

Block | Append | Attach

Block - Replaces the content of the block's name with its most granular reference

Append - Adds the content to the before the most granular block reference

Attach - Adds the content of the block after the most granular reference.
